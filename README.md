# מערכת אזעקות אישית (Red Alert)

מערכת web בזמן אמת להתראות פיקוד העורף, עם תמיכה בבחירת כמה ערים/אזורים, שיוך צליל שונה לכל עיר, ומסך היסטוריה מסונן.

## קישורים רשמיים

- GitHub (Official): [https://github.com/zangiamit91/red-alert](https://github.com/zangiamit91/red-alert)
- Website (Render Demo): [https://red-alert-o5nd.onrender.com](https://red-alert-o5nd.onrender.com)

## מה המערכת כוללת

- חיבור בזמן אמת דרך WebSocket.
- בחירה של כמה ערים/אזורים במקביל.
- צליל ברירת מחדל + צליל נפרד לכל עיר/אזור.
- העלאת קבצי סאונד חדשים (`mp3`, `wav`, `aiff`, `aif`, `m4a`).
- בדיקת התראה לעיר נבחרת או לכל הערים שהוגדרו.
- היסטוריית אזעקות בלבד (ללא התראות בדיקה), כולל סינון:
  - שעות אחרונות (`6h`, `12h`, `24h`)
  - ימים אחרונים (`3d`, `7d`)
  - טווח תאריכים מותאם
- מפה עם סימון אזורים רלוונטיים להתראה.
- תמיכה טובה יותר במובייל (responsive).
- אינדיקציית מצב מקור נתונים (מחובר/חסום).

## התקנה והרצה לוקאלית

דרישות:

- Node.js 22+
- npm

פקודות:

```bash
npm ci
npm start
```

פתיחת האתר:

- [http://localhost:3000](http://localhost:3000)

## מדריך שימוש קצר

1. לחץ על `הפעלת סאונד` פעם אחת אחרי פתיחת האתר (חשוב לדפדפנים).
2. בשדה `עיר / אזור למעקב` כתוב עיר (או בחר מהרשימה) ולחץ `+ הוסף עיר/אזור`.
3. בחר צליל לעיר מתוך הרשימה (או העלה קובץ חדש).
4. לבדיקות:
   - `בדיקת עיר נבחרת`
   - `בדיקת כל הערים (מהגדרות)`
5. בלשונית `היסטוריה` אפשר לסנן לפי שעות/ימים או לפי טווח תאריכים.

## פריסה אמינה באמת (מומלץ)

בפועל, שרתי ענן (כולל Render) נחסמים לעיתים מול מקור פיקוד העורף עם `403`, ולכן היסטוריה/התראות עלולות לא לעבוד בענן.

לשימוש אמין:

- להריץ את השרת על מחשב בישראל (24/7)
- לפתוח פורט מהראוטר או להשתמש ב-DuckDNS

מדריך מלא:

- [DEPLOY_ISRAEL_HOME_HE.md](./DEPLOY_ISRAEL_HOME_HE.md)

סקריפטים מוכנים:

- `scripts/setup_launchd.sh` - הפעלה קבועה ב-macOS
- `scripts/check_stack.sh` - בדיקות health + history
- `scripts/check_features.sh` - בדיקות פיצ'רים (כמה ערים, צליל לכל עיר, היסטוריה בלי בדיקות)
- `scripts/setup_duckdns.sh` - עדכון DNS אוטומטי
- `scripts/start_public_tunnel.sh` - קישור שיתוף זמני

## הערות חשובות (iPhone ומובייל)

- כל עוד האתר פתוח בדפדפן ויש חיבור פעיל, התראות יתקבלו בזמן אמת.
- בדפדפני iPhone אין הבטחה להשמעת סאונד מותאם אישית כשהדף סגור לגמרי או ברקע לאורך זמן.
- אם נדרש אמינות מלאה גם כשהאפליקציה לא פתוחה, צריך פתרון Push/Native App.

## אבטחה והרשאות

- ריפו ציבורי ב-GitHub לא מאפשר לאחרים לדחוף קוד ישירות אליך, אלא אם נתת להם הרשאת כתיבה.
- אחרים יכולים לעשות `fork` ולשנות אצלם, או לפתוח `Pull Request` שאתה מחליט אם למזג.
- כדי למנוע שינויי הגדרות דרך הלינק הציבורי, אפשר להפעיל מפתח ניהול:
  - הגדר משתנה סביבה `ADMIN_KEY` בשרת.
  - ה-UI יציג שדה "מפתח ניהול" ורק עם המפתח אפשר לשנות הגדרות/להעלות סאונד/לשלוח בדיקת התראה.
